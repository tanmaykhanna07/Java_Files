Finally keyword and try-with resources(introduced in java 7)

It's a good practice to use finally, in the example given in the lecture, they created a bufferReader object but didn't close it. Now they can just use another try catch block of code to close the object but what if someone uses return statement in the first try catch block, then it'll not close. So to prevent issues like these we can use the finally keyword and it'll always run doesn't matter if try or catches runs or nah.

Then there's try-with which is basically try(BufferReader bufferReader){

}

what this does is automatically closes the bufferReader without the need of finally and the object needing to be closed is defined in the parenthesis of try.
